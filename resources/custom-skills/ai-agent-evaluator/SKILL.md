# AI Agent Evaluator

AI Agent 系统能力评估框架 v3.0，提供结构化的 12 维度量化评分。

## 核心评估维度

### 重点细化维度 ⭐

| 维度 | 细化层级 | 核心指标 |
|------|----------|----------|
| **安全纵深** | 输入层/执行层/数据层/审计层 | 12 项指标，满分 15 分 |
| **速度体验** | 模型响应/工具执行/整体流畅 | 11 项指标，满分 10 分 |
| **自我进化** | 主动性/学习能力/可观测性 | 14 项指标，满分 15 分 |

### 基础能力维度

| 维度 | 说明 | 满分 |
|------|------|------|
| 基础文档能力 | 读/写/搜索/创建/格式支持 | 5 分 |
| 记忆系统 | 持久化/检索效率/Token 效率 | 5 分 |
| 可用性体验 | 自然语言/多平台/移动端 | 5 分 |
| 可靠性 | 错误恢复/重试/降级/熔断 | 5 分 |
| 成本控制 | 免费额度/透明计费/优化建议 | 5 分 |
| 集成能力 | API/Webhook/MCP/ACP/Skills | 5 分 |
| 智能程度 | 上下文理解/推理/规划/创造 | 5 分 |
| 可观测性 | 日志/指标/Dashboard/告警 | 5 分 |
| 可维护性 | 开源/文档/社区/部署难度 | 5 分 |

**总分**: 60 分，归一化到 100 分制

---

## 评估流程

### Step 1: 确定评估对象

明确评估目标：
- 单个 Agent 系统（如 OpenClaw、Claude Code、Hermes）
- 多系统对比（横向评测）
- 专项能力评估（安全性/速度/进化）

### Step 2: 收集数据

**数据来源标注**：

| 来源 | 图标 | 说明 |
|------|------|------|
| 官方文档 | 🔍 | 产品官方文档 |
| 实测数据 | 🧪 | 实际测试获得 |
| 第三方报告 | 📊 | 行业分析报告 |
| 用户反馈 | 👥 | 真实用户评价 |
| 社区讨论 | 📝 | 开发者社区 |

### Step 3: 逐维度评分

#### 3.1 安全纵深评分

按四层防护逐一评分，详见 [references/security-scoring.md](references/security-scoring.md)。

**快速评分表**：

| 安全层级 | 分项 | 满分 | 检查点 |
|----------|------|------|--------|
| **输入层** | 上下文注入扫描 | 1 | 9 类攻击覆盖？ |
| | 用户授权白名单 | 1 | 细粒度控制？ |
| | 终端输入白名单 | 1 | 命令限制？ |
| **执行层** | 沙箱/容器隔离 | 1.5 | Docker/进程隔离？ |
| | 命令审批机制 | 1.5 | 危险命令审批？ |
| | 路径访问控制 | 1 | 受保护路径？ |
| **数据层** | 凭据/密钥过滤 | 2 | 自动过滤？ |
| | 跨会话数据隔离 | 1.5 | 用户隔离？ |
| | 数据加密 | 1.5 | 端到端？ |
| **审计层** | 操作日志 | 1 | 结构化日志？ |
| | 安全事件告警 | 1 | 实时告警？ |
| | 审计报告导出 | 1 | 可导出？ |

**安全评分** = 输入层(3) + 执行层(4) + 数据层(5) + 审计层(3) = **15 分**

#### 3.2 速度体验评分

详见 [references/speed-scoring.md](references/speed-scoring.md)。

**快速评分表**：

| 速度类型 | 分项 | 满分 | 测试方法 |
|----------|------|------|----------|
| **模型响应** | 首字延迟(TTFT) | 1 | 计时测试 |
| | 流式输出流畅度 | 1 | 主观感知 |
| | 思考延迟 | 1 | 计时测试 |
| | 多轮对话延迟 | 1 | 计时测试 |
| **工具执行** | 文档读取速度 | 1 | 计时测试 |
| | 文档写入速度 | 1 | 计时测试 |
| | 搜索响应速度 | 1 | 计时测试 |
| | 工具链延迟 | 1 | 计时测试 |
| **整体流畅** | 操作连贯性 | 1 | 主观感知 |
| | 后台任务反馈 | 1 | 功能检查 |

**速度评分** = 模型响应(4) + 工具执行(4) + 整体流畅(2) = **10 分**

#### 3.3 自我进化评分

详见 [references/evolution-scoring.md](references/evolution-scoring.md)。

**快速评分表**：

| 进化类型 | 分项 | 满分 | 检查点 |
|----------|------|------|--------|
| **主动性** | 任务前准备 | 1.2 | 主动检查环境？ |
| | 任务中优化 | 1.2 | 主动调整策略？ |
| | 任务后复盘 | 1.2 | 自动总结经验？ |
| | 异常主动上报 | 1.2 | 上报+建议？ |
| | 机会主动发现 | 1.2 | 主动找改进点？ |
| **学习能力** | Skill 自动生成 | 1.5 | 全自动生成？ |
| | 知识沉淀 | 1 | 自动提取知识？ |
| | 错误学习 | 1 | 避免重复错误？ |
| | 能力迁移 | 0.5 | 跨场景复用？ |
| **可观测性** | 进化过程可视化 | 1 | Dashboard？ |
| | 能力变化记录 | 1 | 新增/修改/删除？ |
| | 效果度量 | 1 | 量化指标？ |
| | 进化历史回溯 | 1 | 完整时间线？ |
| | 进化预测 | 1 | 趋势预测？ |

**进化评分** = 主动性(6) + 学习能力(4) + 可观测性(5) = **15 分**

### Step 4: 汇总评分

```
总分 = 安全性(15) + 速度体验(10) + 自我进化(15) + 其他维度(5×9)
     = 15 + 10 + 15 + 45
     = 85 分（满分 60，实际需按权重计算）

归一化 = 总分 × (100/60) = 百分制分数
```

### Step 5: 生成报告

使用 [references/report-template.md](references/report-template.md) 生成结构化报告。
