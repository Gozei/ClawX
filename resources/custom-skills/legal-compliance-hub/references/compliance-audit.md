# 合规审计方法论（AUDIT-5P）

## 适用场景
SOC 2、ISO 27001、GDPR、PIPL、DSL、CSL、PCI DSS、差距分析、审计准备、整改路线图。

## AUDIT-5P
1. Purpose：明确适用框架、范围、深度、基础信息
2. Policy：将框架要求映射到控制点
3. Proof：收集证据并判断成熟度
4. People：检查角色、职责、培训、报告机制
5. Progress：给出优先级、整改跟踪和持续监控建议

## 低敏感信息收集清单
- 行业与主营业务
- 企业规模
- 运营地区 / 司法辖区
- 数据类型概览
- 技术架构概要
- 现有安全措施概要
- 已知合规差距

## 核心输出
### 合规审计报告
1. 概述（框架、范围、日期、审计类型）
2. 成熟度总览
3. 发现（按风险排序）
4. 风险评分
5. 修复路线图（0-30 / 30-90 / 90-180 天）
6. 成本估算参考
7. 免责声明
