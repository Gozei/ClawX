**三、compliance-audit.md — 合规审计方法论**

**合规审计方法论（AUDIT-5P 模型）**

**方法论来源**

综合 ClawHub 上 Compliance Audit Generator、SOC 2 AI Agent
Compliance、Compliance & Audit Readiness Engine、Regulatory Compliance
Audit（均为 @1kalin）的方法论提炼。

**AUDIT-5P 模型总览**

Purpose → Policy → Proof → People → Progress 目的 → 策略 → 证据 → 人员 →
持续改进

**Phase 1: Purpose（目的与范围界定）**

**目标：** 明确审计要回答什么问题。

> 1\. 确定适用框架（参考 \`regulatory-frameworks.md\` 选择）
>
> 2\. 界定审计范围（业务单元、系统、流程）
>
> 3\. 确定审计深度（全面审计 vs 专项审计）
>
> 4\. 收集基础信息

**信息收集清单（低敏感）：**

> • 行业类型和主营业务
>
> • 企业规模（员工数、年收入）
>
> • 业务运营地区/司法管辖区
>
> • 数据类型概览（个人数据/健康数据/支付数据/商业数据）
>
> • 技术架构概要（云服务/自建/混合）
>
> • 现有安全措施概要
>
> • 已知合规差距

**Phase 2: Policy（策略与控制点映射）**

**目标：** 将框架要求映射为具体控制点。

每个框架的核心控制域：

\#### SOC 2（Trust Services Criteria）

|                                 |                          |
|---------------------------------|--------------------------|
|                                 |                          |
| CC1 - Control Environment       | 治理结构、道德准则、承诺 |
| CC2 - Communication             | 信息传递、职责分离       |
| CC3 - Risk Assessment           | 风险识别与响应           |
| CC4 - Monitoring Activities     | 持续监控与修正           |
| CC6 - Logical & Physical Access | 访问控制（逻辑+物理）    |
| CC7 - System Operations         | 变更管理、问题管理       |
| CC8 - Change Management         | 系统变更的授权和测试     |

\#### ISO 27001（Annex A Controls）

|              |     |                            |
|--------------|-----|----------------------------|
|              |     |                            |
| A.5 组织控制 | 37  | 治理、政策、资产、人力     |
| A.6 人员控制 | 8   | 职前/职中/离职             |
| A.7 物理控制 | 14  | 办公场所、设备安全         |
| A.8 技术控制 | 34  | 访问控制、密码学、运营安全 |

\#### GDPR（99 条款 → 核心控制）

|                  |                               |
|------------------|-------------------------------|
|                  |                               |
| 合法性基础       | Art.6 处理目的                |
| 数据主体权利     | Art.15-22 访问/更正/删除/携带 |
| 数据处理者义务   | Art.28 合同要求               |
| 数据保护影响评估 | Art.35 DPIA                   |
| 数据泄露通知     | Art.33-34（72h 内通知）       |
| 跨境传输         | Art.44-49 合法基础            |
| 数据保护官(DPO)  | Art.37-39                     |

\#### 中国法三法联动

|                      |                                        |
|----------------------|----------------------------------------|
|                      |                                        |
| 个人信息保护法(PIPL) | 知情同意、最小必要、跨境传输评估       |
| 数据安全法(DSL)      | 数据分类分级、重要数据目录、风险评估   |
| 网络安全法(CSL)      | 等级保护、网络运营者义务、个人信息保护 |

\#### PCI DSS v4.0（12 个核心要求）

|       |                                        |
|-------|----------------------------------------|
|       |                                        |
| 1-3   | 网络安全（防火墙、密码策略、数据保护） |
| 4-9   | 数据保护（加密、防恶意软件、访问控制） |
| 10-12 | 运营安全（日志、测试、安全策略）       |

**Phase 3: Proof（证据收集与评估）**

**证据类型矩阵：**

|          |                    |                        |
|----------|--------------------|------------------------|
|          |                    |                        |
| 政策文档 | 书面政策和程序     | 信息安全政策、隐私政策 |
| 配置快照 | 系统配置证明       | IAM 规则、网络 ACL     |
| 审计日志 | 操作记录           | 访问日志、变更日志     |
| 认证记录 | 第三方认证         | SOC 2 报告、ISO 证书   |
| 人员记录 | 培训和意识         | 培训记录、签署确认     |
| 测试结果 | 渗透测试、漏洞扫描 | 第三方测试报告         |

**证据成熟度等级：**

|     |                       |           |
|-----|-----------------------|-----------|
|     |                       |           |
| L0  | 无证据                | ❌ 不可信 |
| L1  | 口头声明/非正式记录   | 🔴 低     |
| L2  | 正式文档但未验证      | 🟠 中低   |
| L3  | 正式文档 + 定期审查   | 🟡 中     |
| L4  | 自动化证据 + 持续监控 | 🟢 高     |
| L5  | 第三方独立审计认证    | ✅ 最高   |

**Phase 4: People（人员与角色评估）**

**关键角色检查清单：**

> ☐ 是否指定了合规负责人/DPO？
>
> ☐ 管理层是否对合规有明确承诺？
>
> ☐ 员工是否接受过合规培训？
>
> ☐ 是否有明确的违规报告机制？
>
> ☐ 职责分离是否到位？

**Phase 5: Progress（修复追踪与持续监控）**

**修复优先级矩阵：**

|     |         |          |          |
|-----|---------|----------|----------|
|     |         |          |          |
| 高  | P0-30天 | P1-60天  | P2-90天  |
| 中  | P1-60天 | P2-90天  | P3-180天 |
| 低  | P2-90天 | P3-180天 | P4-待定  |

**持续监控建议：**

|              |      |                |
|--------------|------|----------------|
|              |      |                |
| 访问控制审查 | 季度 | Okta, Azure AD |
| 安全扫描     | 月度 | Qualys, Nessus |
| 策略审查     | 半年 | 内部审计       |
| 合规体检     | 年度 | 第三方审计     |
| 培训与意识   | 季度 | LMS 系统       |
