**🇨🇳 中国法体系**

**个人信息保护法（PIPL）2021.11.01 生效**

**核心原则：**

> • 合法、正当、必要、诚信
>
> • 知情同意（明示同意为原则）
>
> • 最小必要（限于实现处理目的的最小范围）
>
> • 公开透明

**关键合规要求：**

|                      |           |                                    |
|----------------------|-----------|------------------------------------|
|                      |           |                                    |
| 知情同意             | Art.13-14 | 处理前告知，获取单独同意           |
| 个人信息保护影响评估 | Art.55    | 高风险处理前的必做评估             |
| 跨境传输             | Art.38-43 | 安全评估/标准合同/认证 三选一      |
| 个人信息保护官       | Art.52    | 达到数量门槛的必须指定             |
| 数据泄露通知         | Art.57    | 立即通知网信部门和当事人           |
| 自动化决策           | Art.24    | 不得在无合理商业需要时进行差别待遇 |
| 数据携带权           | Art.45    | 提供转移途径                       |

**跨境数据传输三选一：**

> 1\. \*\*数据出境安全评估\*\* → 处理个人信息达 100
> 万人或向境外提供重要数据
>
> 2\. \*\*个人信息出境标准合同（SCC）\*\* → 其他一般情形
>
> 3\. \*\*个人信息保护认证\*\* → 自愿申请

**数据安全法（DSL）2021.9.1 生效**

**核心制度：**

|                  |                                      |
|------------------|--------------------------------------|
|                  |                                      |
| 数据分类分级     | 国家建立分类分级制度，各行业制定细则 |
| 重要数据目录     | 各地区/部门制定重要数据目录          |
| 数据安全风险评估 | 重要数据处理者定期开展风险评估       |
| 数据安全审查     | 涉及国家安全的数据处理活动           |

**网络安全法（CSL）2017.6.1 生效**

**核心义务：**

|                      |                        |
|----------------------|------------------------|
|                      |                        |
| 网络安全等级保护     | 所有网络运营者         |
| 个人信息保护义务     | 所有网络运营者         |
| 关键信息基础设施保护 | 关键信息基础设施运营者 |
| 数据泄露报告         | 网络运营者             |
| 安全事件应急预案     | 网络运营者             |

**🇪🇺 欧盟法体系**

**GDPR（通用数据保护条例）**

**适用条件（任一触发）：**

> • 在欧盟境内设立机构并处理数据
>
> • 向欧盟境内数据主体提供商品/服务
>
> • 监控欧盟境内数据主体行为

**核心数据主体权利：**

|                    |           |                   |
|--------------------|-----------|-------------------|
|                    |           |                   |
| 知情权             | Art.13-14 | 隐私政策/通知     |
| 访问权             | Art.15    | 主体访问请求(SAR) |
| 更正权             | Art.16    | 更正请求          |
| 删除权（被遗忘权） | Art.17    | 删除请求          |
| 限制处理权         | Art.18    | 限制请求          |
| 数据携带权         | Art.20    | 结构化格式导出    |
| 反对权             | Art.21    | 反对处理          |
| 自动化决策权       | Art.22    | 要求人工介入      |

**违规处罚：**

> • 一般违规：最高 1000 万欧元或全球年营收 2%
>
> • 严重违规：最高 2000 万欧元或全球年营收 4%

**AI Act（人工智能法案）2024.8.1 生效**

**AI 系统风险分级：**

|              |                     |              |
|--------------|---------------------|--------------|
|              |                     |              |
| Unacceptable | 禁止的 AI 实践      | 完全禁止     |
| High-Risk    | 安全/基本权利高风险 | 严格合规义务 |
| Limited Risk | 透明度风险          | 披露义务     |
| Minimal Risk | 低风险              | 自愿行为准则 |

**🇺🇸 美国法体系**

**CCPA/CPRA（加州消费者隐私法）**

**消费者权利：**

> • 知情权、删除权、更正权
>
> • 拒绝出售/共享个人信息权
>
> • 不歧视权

**适用门槛：**

> • 年营收 \> 2500 万美元，或
>
> • 处理 ≥ 10 万人个人信息，或
>
> • 年营收 50%+ 来自出售个人信息

**HIPAA（健康保险可携带性和责任法案）**

**适用范围：** 受保护健康信息（PHI）处理者

**核心要求：**

> • Privacy Rule：使用和披露限制
>
> • Security Rule：行政/物理/技术保障
>
> • Breach Notification Rule：泄露通知（60 天内）
>
> • Transaction Rule：电子交易标准

**隐私保护专项（PIA/DPIA 工作流）**

**何时需要进行 PIA/DPIA？**

**中国 PIPL 触发条件：**

> • 处理敏感个人信息
>
> • 利用个人信息进行自动化决策
>
> • 委托处理/向其他处理者提供
>
> • 向境外提供个人信息
>
> • 其他对个人权益有重大影响的处理活动

**GDPR DPIA 触发条件：**

> • 系统性和全面性评估（画像）
>
> • 大规模处理敏感数据
>
> • 大规模公共区域监控
>
> • 新技术的新用途

**PIA/DPIA 标准流程**

1\. 必要性判断 → 是否需要 PIA？2. 描述处理活动 → 数据流图3. 评估必要性 →
目的限制、比例原则4. 识别风险 → 风险来源、受影响群体5. 控制措施评估 →
现有措施是否充分？6. 残余风险接受 → 是否可接受？7. 记录并归档 → PIA
报告8. 持续审查 → 定期更新
