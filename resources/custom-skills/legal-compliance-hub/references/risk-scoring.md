**五、risk-scoring.md — 风险评分体系**

**风险评分体系（RISK-SCORE 矩阵）**

**方法论来源**

综合 ClawHub 上 Compliance Audit Generator（@1kalin）、Regulatory
Compliance Audit（@1kalin）、Enterprise Legal
Guardrails（@lawyered0）的方法论提炼。

**三级风险评分体系**

**Level 1: 发现级（Finding-Level）**

每个具体发现项的风险评分：

**公式：Risk Score = Probability(P) × Impact(I) × Reachable(R)**

\#### 概率（Probability）评分标准

|     |          |                                |
|-----|----------|--------------------------------|
|     |          |                                |
| 5   | 几乎确定 | 已发生或极可能在 90 天内发生   |
| 4   | 很可能   | 历史上有先例，有明确触发条件   |
| 3   | 可能     | 存在多个风险因素               |
| 2   | 不太可能 | 风险因素较少，控制措施部分有效 |
| 1   | 极不可能 | 理论风险，有效控制措施         |

\#### 影响（Impact）评分标准

|     |        |            |                |                   |
|-----|--------|------------|----------------|-------------------|
|     |        |            |                |                   |
| 5   | 灾难性 | \>1000万元 | 大规模负面报道 | 刑事责任/吊销许可 |
| 4   | 重大   | 100-1000万 | 行业负面关注   | 行政处罚/诉讼     |
| 3   | 显著   | 10-100万   | 客户投诉       | 整改要求          |
| 2   | 中等   | 1-10万     | 内部关注       | 书面警告          |
| 1   | 轻微   | \<1万      | 无外部影响     | 无                |

\#### 可触达性（Reachable）评分标准

|     |                              |
|-----|------------------------------|
|     |                              |
| 1.5 | 外部可利用，无身份验证要求   |
| 1.2 | 外部可利用，需要基本身份验证 |
| 1.0 | 内部可利用（已认证用户）     |
| 0.8 | 需要特权访问（管理员/DBA）   |
| 0.5 | 需要物理访问或多因素突破     |

\#### 风险等级映射

|         |          |     |                     |
|---------|----------|-----|---------------------|
|         |          |     |                     |
| 15-37.5 | Critical | 🔴  | 立即（0-7 天）      |
| 8-14.9  | High     | 🟠  | 紧急（7-30 天）     |
| 4-7.9   | Medium   | 🟡  | 计划中（30-90 天）  |
| 1-3.9   | Low      | 🟢  | 常态化（90-180 天） |

**Level 2: 控制域级（Domain-Level）**

按控制域汇总发现，计算域成熟度：

**公式：Domain Score = 100 - (Weighted Finding Sum / Max Possible) ×
100**

\| 控制域 \| Critical \| High \| Medium \| Low \| 域得分 \| 等级
\|\|-------\|----------\|------\|--------\|-----\|-------\|------\|\|
访问控制 \| 1 \| 2 \| 3 \| 1 \| 65 \| 🟡 B \|\| 数据保护 \| 0 \| 1 \| 2
\| 0 \| 75 \| 🟡 B+ \|\| 安全运营 \| 2 \| 0 \| 1 \| 2 \| 55 \| 🟠 C \|

**域成熟度等级：**

|     |        |           |                    |
|-----|--------|-----------|--------------------|
|     |        |           |                    |
| A+  | 95-100 | ✅ 优秀   | 行业领先水平       |
| A   | 85-94  | ✅ 良好   | 完备且运行有效     |
| B+  | 75-84  | 🟡 较好   | 基本完备，少量改进 |
| B   | 65-74  | 🟡 中等   | 有明显差距         |
| C   | 50-64  | 🟠 待改进 | 存在重大差距       |
| D   | \<50   | 🔴 不足   | 需要系统性改造     |

**Level 3: 组织级（Organization-Level）**

综合所有域，计算整体合规成熟度：

**公式：Org Score = Σ(Domain Score × Domain Weight) / Σ(Domain Weight)**

**控制域权重参考：**

|            |     |                          |
|------------|-----|--------------------------|
|            |     |                          |
| 治理与承诺 | 15% | 领导层支持、策略框架     |
| 风险管理   | 15% | 风险评估、应急响应       |
| 访问控制   | 15% | 身份管理、权限管理       |
| 数据保护   | 15% | 加密、备份、数据生命周期 |
| 安全运营   | 10% | 监控、事件管理、变更管理 |
| 人员安全   | 10% | 培训、意识、背景调查     |
| 物理安全   | 5%  | 设施访问、设备管理       |
| 第三方管理 | 10% | 供应商评估、合同要求     |
| 合规审计   | 5%  | 内部审计、外部认证       |
